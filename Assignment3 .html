LIFO Principle O(1) all ops Fixed size (array)

A stack operates on the Last In, First Out principle. The most recently pushed element is always the first to be removed. Every core operation works directly on the top pointer, making all of them constant time.

Q1.1 — Definition

What is a Stack?

A linear structure where insertion and deletion occur only at one end — the top. Follows LIFO: the last item pushed is the first to be popped. Real-world analogies: a stack of plates, browser back-button history, undo in text editors.

Q1.2 — Core Operations

Push · Pop · Peek

Push adds to the top. Pop removes from the top. Peek reads the top without removing. isEmpty checks for an empty stack. All run in O(1) time — they only touch the top pointer.

Q1.3 — Error States

Overflow & Underflow

Stack Overflow — pushing onto a full stack. Occurs during push. Stack Underflow — popping from an empty stack. Occurs during pop. Robust code must handle both gracefully with error messages.

Q1.4 — Implementation

Array vs Linked List

Array: Fixed memory, cache-friendly, fast. Cannot resize after declaration. Linked List: Dynamic growth, no size limit, but each node carries an extra pointer overhead. Arrays are faster; linked lists are more flexible.

Stack Implementation

Array-based & Linked-list-based in C++

push → O(1) pop → O(1) peek → O(1) display → O(n)
Array Stack — C++ 
class Stack {
  int arr[10];
  int top;
public:
  Stack() { top = -1; }

  bool isEmpty() { return top == -1; }
  bool isFull()  { return top == 9; }

  void push(int val) {
    if (isFull()) { cout << "Stack Overflow"; return; }
    arr[++top] = val;
  }

  int pop() {
    if (isEmpty()) { cout << "Stack Underflow"; return -1; }
    return arr[top--];
  }

  int peek() { return isEmpty() ? -1 : arr[top]; }
};
Linked List Stack — C++ 
class StackLL {
  Node* top;
public:
  StackLL() { top = NULL; }

  void push(int val) {
    Node* newNode = new Node(val);
    newNode->next = top;   // point to old top
    top = newNode;           // update top
  }

  int pop() {
    if (isEmpty()) return -1;
    int val = top->data;
    Node* temp = top;
    top = top->next;
    delete temp;             // free memory
    return val;
  }
};

Balanced Brackets Checker

Stack Application · O(n) Time · O(n) Space

Algorithm: Traverse the expression left to right. Push opening brackets onto the stack. For each closing bracket, check if it matches the stack's top. If yes, pop it. If at the end the stack is empty — the expression is balanced.

bool isBalanced(string expr) {
  stack<char> s;
  for (char ch : expr) {
    if (ch == '(' || ch == '[' || ch == '{')
      s.push(ch);
    else if (ch == ')' || ch == ']' || ch == '}') {
      if (s.empty()) return false;
      if ((ch==')' && s.top()=='(') ||
          (ch==']' && s.top()=='[') ||
          (ch=='}' && s.top()=='{'))
        s.pop();
      else return false;
    }
  }
  return s.empty(); // empty = balanced
}

Q4.1 — Definition

What is a Queue?

Insertion at the rear, deletion at the front. FIFO principle. Real-world: bank queues, print job scheduling, CPU task scheduling.

Q4.2 — Operations

Enqueue · Dequeue · Front

Enqueue inserts at rear. Dequeue removes from front. Front peeks without removing. All run in O(1) by directly accessing front/rear pointers.

Q4.3 — Circular Queue

Why Circular?

Last position wraps back to first using the modulo operator. Empty front slots left by dequeues can be reused, solving the "false full" problem of linear array queues.

Q4.4 — Implementation

Array vs Linked List

Array queues are fast but fixed-size and may report overflow when front slots are empty. Linked list queues grow dynamically and use memory only as needed, making them generally preferred.

Queue Implementation

Linked List Queue & Circular Array Queue in C++

Linked List Queue — C++ 
class Queue {
  Node *front, *rear;
public:
  Queue() { front = rear = NULL; }

  void enqueue(int val) {
    Node* n = new Node(val);
    if (isEmpty()) { front = rear = n; return; }
    rear->next = n;
    rear = n;          // advance rear
  }

  int dequeue() {
    if (isEmpty()) return -1;
    int val = front->data;
    Node* tmp = front;
    front = front->next;  // advance front
    delete tmp;
    return val;
  }
};
Circular Array Queue — C++ 
class CircularQueue {
  int arr[6], front, rear;
public:
  CircularQueue() { front = rear = -1; }

  bool isFull()  { return (rear+1)%6 == front; }
  bool isEmpty() { return front == -1; }

  void enqueue(int val) {
    if (isFull()) { cout << "Overflow"; return; }
    if (isEmpty()) front = rear = 0;
    else rear = (rear+1)%6;   // wrap around
    arr[rear] = val;
  }

  void dequeue() {
    if (front == rear) front = rear = -1;
    else front = (front+1)%6; // wrap around
  }
};

Evaluate Postfix Expression

Stack Application · Expression Parsing

Algorithm: Scan left to right. If digit → push. If operator → pop two operands (val1, val2), compute val1 OP val2, push result. The final stack element is the answer.

evaluatePostfix — C++ 
int evaluatePostfix(string expr) {
  stack<int> s;
  for (char ch : expr) {
    if (isdigit(ch)) {
      s.push(ch - '0');   // char to int
    } else {
      int val2 = s.top(); s.pop();
      int val1 = s.top(); s.pop();
      switch(ch) {
        case '+': s.push(val1 + val2); break;
        case '-': s.push(val1 - val2); break;
        case '*': s.push(val1 * val2); break;
        case '/': s.push(val1 / val2); break;
      }
    }
  }
  return s.top();
}
